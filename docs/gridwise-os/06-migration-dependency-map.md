## Rollback order
Reverse of the DAG. Every migration file has a sibling `*_rollback.sql`. Rollbacks are non-destructive to pre-Phase-1 columns/data вҖ” only Phase-1+ additions are dropped.

## Cycle-freeness
Verified by hand: no back-edges. `partners вҶ’ wp_partner_allocations вҶ’ work_packages` is a straight chain; `dno_offer_sites`/`po_line_sites` depend on both parents but never on `sites` in the other direction.
